# WooCamers Helper

افزونه وردپرس برای تولید خودکار ویژگی‌ها، توضیح مختصر و توضیحات محصولات ووکامرس با استفاده از هوش مصنوعی.

**نسخه:** 0.3.3

---

## امکانات

- **تولید ویژگی محصول با AI** — با یک کلیک، ویژگی‌های پیشنهادی برای هر محصول بر اساس اطلاعات موجود تولید می‌شود.
- **تولید توضیح مختصر (Excerpt)** — متن کوتاه و روان برای فیلد توضیح مختصر محصول.
- **تولید توضیحات کامل محصول** — متن قانع‌کننده برای بدنه اصلی محصول.
- **پشتیبانی از سه حالت اتصال:** آزمایشی (Mock)، OpenAI / Gap GPT، و API سفارشی.
- **پرامپت قابل تنظیم** — هر سه پرامپت را از پنل تنظیمات ویرایش کنید.
- **نمایش هزینه تقریبی** — هزینه هر درخواست به دلار و تومان نمایش داده می‌شود.
- **ویرایش ویژگی‌ها قبل از ذخیره** — در مودال نتیجه می‌توانید نام و مقدار هر ویژگی را ویرایش کرده یا تیک آن را بردارید.

---

## پیش‌نیازها

- WordPress 5.8 یا بالاتر
- WooCommerce 6.0 یا بالاتر
- PHP 7.4 یا بالاتر
- (برای حالت AI واقعی) کلید API از OpenAI یا سرویس سازگار با Chat Completions

---

## نصب

1. پوشه `woocamers-helper` را در `/wp-content/plugins/` کپی کنید.
2. از پیشخوان وردپرس به **افزونه‌ها → افزونه‌های نصب‌شده** بروید.
3. افزونه **WooCamers Helper** را فعال کنید.

---

## راه‌اندازی

پس از فعال‌سازی، از منوی **WooCommerce → Helper WooCamers** به صفحه تنظیمات بروید.

### تنظیمات اصلی

| فیلد | توضیح |
|---|---|
| **حالت اتصال** | نحوه ارتباط با AI را مشخص می‌کند (توضیح در زیر) |
| **API Endpoint** | آدرس API سفارشی یا OpenAI-compatible |
| **API Key** | کلید API (برای حالت Gap GPT اجباری است) |
| **نرخ دلار به تومان** | برای تبدیل هزینه API به تومان — اگر صفر باشد فقط هزینه دلاری نمایش داده می‌شود |
| **مدل OpenAI** | نام مدل، مثل `gpt-4o` یا `gpt-4o-mini` |
| **Reasoning effort** | سطح استدلال مدل: `none`, `low`, `medium`, `high` |

### حالت‌های اتصال

**آزمایشی / محلی (Mock)**
هیچ درخواستی به API ارسال نمی‌شود. یک پاسخ نمونه ثابت برمی‌گردد. برای تست ظاهر و جریان کار مناسب است.

**Gap GPT / OpenAI Chat Completions**
درخواست به فرمت Chat Completions ارسال می‌شود. اگر Endpoint خالی باشد، آدرس پیش‌فرض `https://api.openai.com/v1/chat/completions` استفاده می‌شود. وارد کردن API Key اجباری است.

**API سفارشی (Endpoint)**
درخواست به آدرس دلخواه شما ارسال می‌شود. ساختار درخواست JSON شامل فیلدهای `mode`، `task`، `prompt` و `product` است.

### پرامپت‌ها

سه پرامپت جداگانه برای هر عملیات وجود دارد. در متن پرامپت می‌توانید از جایگزین‌های زیر استفاده کنید که به‌صورت خودکار با اطلاعات محصول پر می‌شوند:

| متغیر | توضیح |
|---|---|
| `{title}` | نام محصول |
| `{excerpt}` | توضیح مختصر فعلی |
| `{description}` | توضیحات کامل فعلی |
| `{price}` | قیمت |
| `{categories}` | دسته‌بندی‌ها |
| `{tags}` | برچسب‌ها |
| `{sku}` | SKU محصول |
| `{attributes}` | ویژگی‌های موجود |
| `{current_short_description}` | توضیح مختصر فعلی (برای پرامپت توضیح مختصر) |
| `{current_description}` | توضیحات کامل فعلی (برای پرامپت توضیحات محصول) |

## نحوه استفاده

### تولید ویژگی

سه روش برای دسترسی به این قابلیت وجود دارد:

1. **لیست محصولات** — ستون «Helper» با دکمه «افزودن ویژگی» در کنار هر محصول.
2. **ردیف محصول** — در منوی Row Actions هر محصول لینک «افزودن ویژگی» نمایش داده می‌شود.
3. **ویرایش محصول** — در تب «ویژگی‌ها» دکمه «افزودن ویژگی» وجود دارد.

پس از کلیک، یک مودال باز می‌شود که:
- پرامپت ارسالی را نشان می‌دهد
- هزینه تقریبی را نمایش می‌دهد
- ویژگی‌های پیشنهادی را با فیلدهای قابل ویرایش نشان می‌دهد
- با دکمه «افزودن ویژگی‌ها» فقط ردیف‌های تیک‌خورده ذخیره می‌شوند

### تولید توضیح مختصر

در صفحه ویرایش محصول، کنار فیلد «توضیح مختصر» دکمه «تولید متن با هوش مصنوعی» ظاهر می‌شود. پس از کلیک، متن تولیدشده مستقیماً در فیلد درج می‌شود.

### تولید توضیحات کامل

در صفحه ویرایش محصول، کنار ویرایشگر اصلی محصول دکمه «تولید متن با هوش مصنوعی» ظاهر می‌شود. نتیجه در ویرایشگر (کلاسیک یا TinyMCE) درج می‌شود.

## پشتیبانی از API سفارشی

اگر از API سفارشی استفاده می‌کنید، افزونه یک درخواست POST با این ساختار JSON ارسال می‌کند:

```json
{
  "mode": "endpoint",
  "task": "attributes | short_description | product_description",
  "prompt": "...",
  "product": {
    "title": "...",
    "excerpt": "...",
    "description": "...",
    "price": "...",
    "categories": ["..."],
    "sku": "...",
    "tags": "...",
    "attributes": "...",
    "current_description": "...",
    "current_short_description": "..."
  }
}
```

برای `task: attributes`، پاسخ باید JSON با این ساختار باشد:

```json
{
  "attributes": [
    { "name": "رنگ", "value": "قرمز" },
    { "name": "وزن", "value": "500 گرم" }
  ]
}
```

برای `task: short_description` و `task: product_description`، پاسخ می‌تواند متن ساده یا JSON با فیلد `choices[0].message.content` باشد (فرمت OpenAI).
